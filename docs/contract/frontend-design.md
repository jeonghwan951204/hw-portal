# 동스크랩 계약 관리 — 프론트엔드 화면 설계

프론트엔드에서 구현할 화면·컴포넌트·인터랙션만 다룬다.
계산·저장·검증 등은 **서버가 처리**하므로, 프론트는 "요청 보내고 결과 표시"만 한다.
(데이터 구조·계산 로직은 별도 문서 참조)

---

## 0. 공통 규칙

### 반응형
- 카드(블록) 단위로 설계. PC는 여러 열, 모바일은 1열로 재배치.
- 목록·상세·거래 모두 카드형 기반.

### 표시 포맷
- 날짜: `yyyy.MM.dd` (예: 2025.07.01).
- 숫자: 천 단위 콤마 (예: 11,998).
- 단가 단위 표기: 목록은 항상 `원/kg`. 상세는 계약 통화·단위 그대로(`USD/ton` 등).

### 프론트가 하는 것 / 안 하는 것
- **한다**: 입력값 모으기, 서버에 요청, 응답 표시, 화면 상태 전환, 로딩/빈 상태
  표시, 확인 모달/토스트, 조건부 노출(정산가 숨김 등).
- **안 한다**: 단가·정산 금액 계산, 정렬·검색·페이지네이션 로직, 유효성 최종 판단,
  단가 확정 여부 결정 → 전부 서버 결과를 받아서 표시만.
  (음수 입력 막기 같은 즉각적 입력 제약 정도는 프론트에서 가능하나, 최종 검증
  오류 메시지는 서버가 보낸 것을 표시)

### 확인 모달 / 토스트
- **확인 모달**(예/아니오, 커스텀 컴포넌트 — 브라우저 alert 사용 안 함):
  중요한 데이터 조작에만 → **계약 등록·수정·삭제**. 삭제는 경고 스타일(빨강).
- **바로 실행 + 토스트**: 딸린 것·연속 입력 → **거래 추가, 결제 입력**.
  매번 확인창 띄우지 않음.
- **토스트**: 성공("저장되었습니다") / 오류(서버가 보낸 메시지) 표시.

### 상태 표시
- 빈 상태: "등록된 계약이 없습니다" 등 안내 문구 + (해당 시) 등록 버튼.
- 로딩 상태: "로딩중" 메시지. 재계산 등 오래 걸리는 작업에 표시.
- 계약 상태 배지: 진행중 / 완료 (값은 서버에서 받음).

## 1. 계약 목록 화면

### 레이아웃 (위 → 아래)
1. **시장 현황 배너**: 최근 LME(USD/ton) + 환율(원/USD) 카드 2개.
   각각 **기준 날짜 함께 표시**(두 날짜가 다를 수 있음). 값은 서버에서 받아 표시.
2. **툴바**: 상태 필터(진행중/완료/전체, 기본 **전체**) · 계약명 검색 입력 ·
   [계약 등록] 버튼 · [재계산] 버튼.
   - 필터·검색: 입력만 받아 서버에 요청, 결과 목록 표시(프론트가 거르지 않음).
   - **재계산 버튼**: 클릭 → 서버에 재계산 요청 → **로딩 표시**(시간 걸림) →
     완료 시 목록 갱신 + 토스트.
3. **계약 카드 그리드**: PC 2~3열, 모바일 1열.
4. **페이지네이션**: 페이지당 10건. 페이지 클릭 → 서버에 해당 페이지 요청.

### 계약 카드 구성
- **크게**: 계약명, 회사명, 상태 배지, 단가 값(원/kg).
- **작게**: 계약기간(`yyyy.MM.dd – MM.dd`), 각 단가의 산정기간·요율.
- **단가 영역**: 대표 품목의 입력된 단가를 유형별로 세로 나열.
  각 줄 = `[유형]  [산정기간·요율(작게)]  ……  [값] 원/kg`.
  - 유형: 가단가/확정가/정산가/원가 등(계약마다 개수 다름 → 있는 만큼 렌더).
  - 요율은 **단가 줄마다** 표시(단가별로 다를 수 있음, 대표 품목 기준).
  - **품목명은 카드에 표시 안 함**.
  - **정산가 줄은 확정가가 확정된 경우에만 렌더**(미확정이면 숨김).
- 카드 클릭 → 계약 상세로 이동.

### 빈/로딩
- 계약 없음: "등록된 계약이 없습니다" + [계약 등록].
- 로딩중: "로딩중" 표시.

## 2. 계약 상세 화면

**탭 2개**. 조회 + 거래 입력 위주.

### 계약 헤더 (탭 위 고정)
- 계약번호 + 상태 배지, 회사명 · 계약기간.
- 통화·단위 문구는 **표시 생략**.
- [수정] 버튼(등록 화면 재사용), [삭제] 버튼(확인 모달).

### 탭 1 — 계약·단가
- **단가별 기간 줄**: 각 단가를 한 줄씩. `[유형] [기간] [상태]` + 오른쪽에
  **기준값(LME·환율)**. 수출이면 LME만, 내수면 LME+환율(계약 유형에 따라 조건부).
  값은 서버에서 받아 표시.
- **품목 표**: 행=품목, 열=단가유형별 값 + 품목 옵션(요율·프리미엄).
  대표 품목 강조. 값은 계약 통화·단위(서버가 계산한 값 표시).
- 일별 추이는 표시 안 함.

### 탭 2 — 거래 내역
- **표**: 날짜 · 품목 · 단가유형 · 수량 · 단가 · 총금액 · 결제여부.
  (모바일은 카드형)
- **거래 바로 입력**(확인 모달 없이, 저장 후 토스트):
  - 표 하단 입력 행(모바일은 [거래 추가] 버튼 → 폼).
  - 입력: 날짜 · 품목 · 단가유형 · 수량. **단가·금액은 서버가 계산해 반환** → 표시.
  - 단가유형 드롭다운: **정산가는 확정가 확정 시에만** 항목으로 노출.
- **결제 입력**(확인 모달 없이, 저장 후 토스트):
  - 미결제 행 클릭 → 그 자리에서 결제 입력 영역 펼침(펼침/접힘 토글).
  - 계약 통화에 따라 폼 전환:
    - 내수(원화): 실입금액 · 입금일 입력. (정산금액 대비 차액은 표시만)
    - 수출(외화): 수취 외화 · 환전 환율 · **원화 입금액** 모두 직접 입력.
      참고 표시(입력 아님): 달러 대조 표시, 원화 차액.
  - 원화 입금액은 **자동계산하지 않음**(직접 입력). 참고용 대조만 표시.

## 3. 계약 등록 / 수정 화면

**스텝형**. 수정은 기존 값 채워 재사용. 최종 저장 시 **확인 모달**.

### 스텝 진행 표시
```
1 기본  →  2 단가  →  3 품목  →  4 확인
```
(단가를 품목보다 먼저 — 단가 개수가 품목 요율 칸 수를 결정)

### 스텝 1 — 계약 기본
- 입력: 계약번호, 거래처, 시작일/종료일, 거래구분(수출/내수), 단가 단위(TON/KG).
- [다음].

### 스텝 2 — 단가 (추가형)
- '단가 추가'로 카드 0~N개(가단가/확정가/정산가/원가). 유형 선택.
- 각 카드: 산정 시작/종료, 계산식 선택. (고정값 계산식이면 고정 단가 입력란 노출)
- [삭제] 각 카드. [이전]/[다음].
- **여기서 추가한 단가 목록이 스텝 3 품목 테이블의 요율 컬럼을 만든다.**

### 스텝 3 — 품목 (테이블 · 행 추가)
- 한 행 = 품목. [품목 추가]로 행 추가, 각 행 [삭제].
- 컬럼: 품목명 · **단가별 요율 칸(스텝 2 단가 수만큼 동적 생성)** · 대표(라디오).
- **프리미엄 기본 숨김**: 행의 [+프리미엄] 클릭 시 그 행 아래로 단가별 프리미엄
  입력 펼침. 안 펼치면 프리미엄 값 없음(빈 0 저장 아님).
- **대표 품목**: 첫 번째 품목이 기본 선택. 라디오로 변경(하나만).
- [이전]/[다음].

### 스텝 4 — 확인
- 입력 요약 표시 → [등록/저장] → **확인 모달**("저장하시겠습니까?") → 서버 요청.
- 서버 검증 오류 시 해당 메시지 토스트로 표시(계약번호 중복 등).

### 입력 제약(프론트 즉시)
- 수량·단가 등 음수 입력 막기(0은 허용). 그 외 최종 검증은 서버.

---

## 4. 삭제 인터랙션

- 상세(또는 목록)에서 [삭제] → **확인 모달**(경고 스타일, "삭제하시겠습니까?
  목록에서 숨겨집니다") → 서버에 삭제 요청 → 결과 토스트.
- 실제 삭제 처리(soft delete, 연쇄 등)는 서버. 프론트는 요청+결과 표시만.

## 화면 흐름
```
목록 ──클릭──▶ 상세(탭1 계약·단가 / 탭2 거래내역)
 │  ├─[계약 등록]──▶ 등록 스텝(1~4) ──저장──▶ 목록
 │  └─[재계산]──▶ (로딩) ──▶ 목록 갱신
 └─ 상세 ─[수정]──▶ 등록 스텝(값 채움) / ─[삭제]──▶ 확인 모달
```

---

## 컴포넌트 재사용 메모
- **확인 모달**: 등록/수정/삭제 공용(제목·문구·버튼색만 다르게).
- **토스트**: 성공/오류 공용.
- **계약 카드**: 목록에서 반복. 단가 줄은 유형 배열을 반복 렌더(개수 가변).
- **단가별 요율 입력 그리드**: 등록 스텝 3에서 단가 배열 기반 동적 컬럼.
- **결제 입력 폼**: 내수/수출 두 변형(계약 통화로 분기).

- 목록 정렬 UI 노출 여부(정렬 자체는 서버).
